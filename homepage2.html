Aprendiendo sobre XML, JSON Y AJAX



DIFERENCIAS XML JSON
1 Lenguaje de Marcas Lenguaje de Intercambio de Datos
2 Tamaño de archivo más grande, más detallado, más lento Tamaño de archivo más pequeño, menos detallado, más rápido
3 Es más difícil de analizar, requiere un analizador XML Fácil de analizar con la función JS simple, listo para usar como objeto JS
4 Uso con API SOAP y REST Uso con API REST
5 Soporta varios formatos de codificación Solo admite codificación UTF-8 y UTF-16
6 Los datos no tienen tipo Los datos tienen tipo (cadena, booleano, número, matriz, objeto, nulo)
4 Admite esquemas, espacios de nombres y metadatos No es compatible con el esquema, el espacio de nombres y los metadatos.
4 Estructura de datos de árbol (jerárquica) Estructura de datos del mapa (claves y pares de valores)
4 No admite matrices Matrices de soporte






SEMEJANZAS XML VS JSON
1 Tanto JSON como XML son formatos autodescriptivos
Los formatos autodescriptivos se diseñaron para que fueran legibles para el ser humano, legibles para el ser humano, legibles para la máquina y legibles para la máquina.
2 Tanto JSON como XML tienen un buen soporte para las definiciones de contenido y la validación
JSON y XML son ampliamente utilizados, por lo que son soportados por muchos lenguajes de programación. Por ejemplo, JSON es compatible con JavaScript, Python, Perl y Ruby. XML data también es compatible con muchos lenguajes de programación diferentes: JavaScript, PHP y C# (que es la versión de Microsoft de C++).
3 La serialización de datos
JSON y XML pueden utilizarse para la serialización de datos (convertir una estructura de datos a un formato para su almacenamiento o transmisión).
4 Basado en texto
Una de las similitudes más evidentes entre estas dos estructuras de datos es que JSON y XML están basadas en texto. Por ello, muchos creen que JSON se creó para ser una alternativa más sencilla a XML.
5 Estructuras jerárquicas
En tercer lugar, ambos formatos tienen estructuras jerárquicas en las que cada campo tiene su nombre y valor separados por comas.


Que es AJAX?

Definición:

AJAX son las siglas de Asynchronous JavaScript And XML, (Javascript asíncrono y XML). No es en sí un lenguaje de programación, sino una nueva técnica que combina varios lenguajes de programación.




Componentes de AJAX


Ajax es una combinación de los siguientes lenguajes de programación y elementos:


HTML (o XHTML) y CSS: Base para el diseño de la página.


DOM y Javascript: Forma de acceder dinámicamente a las partes de la página.


objeto XMLHttpRequest: Es el que permite la comunicación asíncrona (en segundo plano) con el servidor.


XML : Formato en el que están los datos que se solicitan al servidor; aunque otros formatos también pueden funcionar, como son HTML, texto plano(txt), json ... etc.


PHP : En este manual trataremos también cómo mandar datos al servidor. Este los recoge mediante PHP. Una vez enviados pueden guardarse en una base de datos o procesarlos para enviar alguna información.
